# GTEC Boys Hostel — মিল ম্যানেজমেন্ট (ধাপ ১: ফাউন্ডেশন)

এই ধাপে যা বানানো হয়েছে:
- সাইনআপ/লগইন (ইউজারনেম + পাসওয়ার্ড, Firebase Auth দিয়ে)
- ইউজার অ্যাপ্রুভাল সিস্টেম (pending → approved/rejected)
- অ্যাডমিন ড্যাশবোর্ড: পেন্ডিং লিস্ট, Approve All, রুম-ওয়াইজ ইউজার লিস্ট, ইউজার ডিলিট
- রোল-বেসড রাউটিং (student vs admin আলাদা পেজে যাবে)
- PWA manifest (বেসিক)

---

## Firebase Console এ যা করতে হবে (একবার মাত্র)

### ১. Email/Password Authentication চালু করো
Firebase Console → Authentication → Sign-in method → **Email/Password** → Enable করো।

### ২. Firestore Database চালু করো
Firebase Console → Firestore Database → Create database → **Production mode** সিলেক্ট করো (Test mode না, কারণ আমরা নিজেদের rules বসাবো)।

### ৩. Security Rules বসাও
Firestore Database → Rules ট্যাবে গিয়ে `firestore.rules` ফাইলের পুরো কন্টেন্ট কপি-পেস্ট করে **Publish** করো।

### ৪. নিজেকে মেইন অ্যাডমিন বানাও
এই ধাপে সব নতুন সাইনআপ ডিফল্টে "student" + "pending" হয় (সিকিউরিটির জন্য ইচ্ছাকৃতভাবে)। তাই প্রথম অ্যাডমিন অ্যাকাউন্ট বানাতে ম্যানুয়ালি করতে হবে:

1. সাইটে গিয়ে নিজের জন্য একটা অ্যাকাউন্ট সাইনআপ করো (তোমার নাম, রুম নাম্বার, ইউজারনেম, পাসওয়ার্ড দিয়ে)।
2. Firebase Console → Firestore Database → `users` কালেকশনে গিয়ে তোমার ডকুমেন্টটা খুঁজে বের করো (নাম দিয়ে চেনা যাবে)।
3. সেই ডকুমেন্টে দুইটা ফিল্ড ম্যানুয়ালি এডিট করো:
   - `status`: `"pending"` থেকে `"approved"` করো
   - `role`: `"student"` থেকে `"mainadmin"` করো
4. এখন লগআউট করে আবার লগইন করলে সরাসরি অ্যাডমিন ড্যাশবোর্ডে ঢুকবে।

এরপর থেকে বাকি সবাইকে (সাব-অ্যাডমিন সহ) তুমি নিজের অ্যাডমিন প্যানেল থেকেই ম্যানেজ করতে পারবে।

## GitHub Pages এ পাবলিশ করা

1. এই পুরো ফোল্ডারের কন্টেন্ট `e1websitesell/gtec` রিপোজিটরিতে পুশ করো।
2. GitHub রিপোতে যাও → Settings → Pages
3. Source: **Deploy from a branch**, Branch: `main` (অথবা যে ব্রাঞ্চে কোড আছে), folder: `/ (root)`
4. Save করলে কিছুক্ষণের মধ্যে একটা লিংক পাবে (যেমন `https://e1websitesell.github.io/gtec/`)

- মিল টগল, বিলিং, রিপোর্ট, পোল ইত্যাদি — এগুলো পরের ধাপে আসবে।
